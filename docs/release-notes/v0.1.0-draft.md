# Lumi v0.1.0 Draft Release Notes

Status: draft for V1 validation.

## Summary

Lumi v0.1.0 is the first Emby-first desktop validation build. It focuses on one complete viewing loop: connect an Emby server manually, browse media, open details, play through native libmpv, and report playback progress back to Emby.

## Highlights

- Tauri 2 desktop shell with React 19.
- Home rails for Continue Watching, Media Libraries, latest media, and featured media.
- Read-only Favorites page for the selected Emby server.
- Media detail pages for movies, series, seasons, episodes, videos, and common container entries.
- Runtime-loaded native libmpv playback through the Rust `PlayerService`.
- Playback commands for play, pause, seek, volume, and close.
- Emby progress reporting during playback and final stopped reporting on close.
- SQLite-backed server profiles and media cache.
- System keyring token storage.
- Settings for servers, player preferences, appearance/material fallback state, language, and diagnostics export.

## Platform Requirements

- Windows 11 is the primary V1 desktop validation target.
- macOS is a V1 validation target, but bundle constraints must be checked before release.
- A reachable Emby server is required for real V1 validation.
- libmpv is required for real playback validation.

## libmpv Distribution

Lumi dynamically loads libmpv so the library remains replaceable. The app checks `LUMI_LIBMPV_PATH`, system loader paths, app-local libraries, bundled resources, and the local development resource folder.

Audited manifest entries currently exist for:

- `windows-x64`: `zhongfly/mpv-winbuild mpv-dev-lgpl x86_64 2026-05-07`
- `windows-arm64`: `zhongfly/mpv-winbuild mpv-dev-lgpl aarch64 2026-05-07`

Both entries are recorded in `scripts/libmpv-sources.json` with pinned URLs, SHA256 hashes, source URLs, and `LGPL-2.1-or-later` metadata.

macOS libmpv bundle sources are not yet audited in the manifest. Do not publish a macOS bundle that silently includes an unreviewed libmpv binary.

## License Notice

mpv/libmpv is used as a replaceable dynamic library. Release artifacts that bundle libmpv must include the libmpv source metadata produced by `scripts/download-libmpv.ps1` and a third-party license notice for the exact binary source.

Do not bundle GPL-constrained mpv builds into default Lumi releases without an explicit release decision and notice.

## Known Limits

- V1 supports Emby only.
- Favorites are read-only and scoped to the selected server.
- Search is present as an app section but is not a complete cross-library search experience.
- Material state currently reports fallback probing until Rust platform detection is implemented.
- macOS vibrancy/Liquid Glass behavior is pending manual validation.
- Real playback requires libmpv to be installed, staged, or provided through `LUMI_LIBMPV_PATH`.
- Real progress sync requires manual validation against an Emby server.

## Validation

Use `docs/validation/v1-acceptance.md` as the release gate checklist. Automated checks must pass, and manual Emby playback plus platform checks must be completed before marking v0.1.0 ready.
